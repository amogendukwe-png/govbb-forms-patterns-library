How to use this library when working on a new form

1

Run a pattern check before writing anything new. When a new form comes in, search this library for fields that look similar. If the form has a name, address, or contact section — it's P-01, P-02, P-04. Reference those by ID in your spec rather than re-describing them. This is what makes specs faster over time.

2

Use this AI prompt to generate a pattern delta for any new form. Paste it into Claude with the form attached:

You are a service designer working on the GovTech Barbados forms prototyping team. Analyse the attached form against the GovTech Barbados Pattern Registry (P-01 to P-37). The registry covers two MDAs: NISSS (P-01 to P-24) and CAIPO (P-25 to P-37). Note the MDA context of the form you are analysing — NISSS patterns may appear in CAIPO forms and vice versa. For each section of the form, output: 1. MATCHED — pattern ID + name (e.g. "P-02 Barbados Address Block") + note if this is cross-MDA 2. VARIANT — pattern ID + how this instance differs from the standard (e.g. "P-04 Contact Block — Fax field added") 3. NEW — a proposed pattern name, description, and field list not covered by any existing pattern Then output a FORM COMPLEXITY ASSESSMENT using the Form Categorisation criteria (Simple / Medium / Complex). Finally, list any OPEN QUESTIONS that should be added to the MDA question log, tagged with the relevant MDA.
3

Update the registry after each new form. For each NEW pattern found, add it to the spreadsheet (NISSS_Pattern_Registry.xlsx) — noting the MDA it was found in and this HTML file with status 🔴 New. When a pattern reaches 2 occurrences, update to 🟡 Emerging. At 3, promote to 🟢 Standard. Update the occurrence counter on each existing pattern card.

4

Connect decisions to patterns. When you log a design decision in the Decision Log that affects a pattern, add the Pattern ID as a tag (e.g., P-06, P-02). The "View decisions" link on each pattern card above will then surface all relevant decisions automatically. Also add the Question ID (e.g., Q-01) as a tag when a question is answered so the log becomes searchable by question.

5

Use the Open Questions table as your MDA prep. Before each MDA session, filter by the form being discussed and share the relevant rows. Answers come back → log in the Decision Log with the Question ID as a tag → mark the question Status as Resolved in this file.
